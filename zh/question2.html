# 常见问题解答

# Aca.ts 如何连接多个数据库？

可以在一个项目中同时连接多个数据库，在数据模型的定义文件中，我们使用顶级命名空间来对应不同的数据库

example - blog.ts

namespace Blog {
  const dbConfig: $db = 'default'
}
namespace Test {
  const dbConfig: $db = 'test'
}

config.json

  "databases": {
    "default": {
      "tableNameWithNamespace": true,
      "onlyApi": false,
      "idDefaultType": "cuid",
      "foreignKeyConstraint": true,
      "onUpdate": "cascade",
      "onDelete": "cascade",
      "connectOption": {
        "driver": "mssql",
        "envConnect": "",
        "connect": {
          "host": "",
          "port": ,
          "user": "",
          "password": "",
          "database": ""
        }
      }
    },
    "test": {
      "tableNameWithNamespace": true,
      "onlyApi": false,
      "idDefaultType": "cuid",
      "foreignKeyConstraint": true,
      "onUpdate": "cascade",
      "onDelete": "cascade",
      "connectOption": {
        "driver": "pg",
        "envConnect": "",
        "connect": {
          "host": "",
          "port": ,
          "user": "",
          "password": "",
          "database": ""
        }
      }
    }
  }

# Aca.ts 是如何实现在前端写后端代码？

Aca.ts 能够根据定义的数据模型文件，自动生成前后端访问数据库的 API 接口。开发者可以在任何前端框架中使用 API 访问数据库。不需要编写复杂的 AJAX 请求方法，不需要定义复杂的路由接口。在前端编写的代码也可以直接复制到后端运行。

前端

import { $RPC, Blog } from './aca.client'

await Blog.user.insert({
  data: {
    firstName: 'foo',
    lastName: 'bar',
  },
})

const rtn = await $RPC.server.example({ firstName: 'foo', lastName: 'bar' })

# 运行在后端的 npm 包，如何在前端代码中使用？

如下图所示，在后端编写的函数，能够自动生成同名的前端 API 方法。可以在前端中直接使用

后端

import { v4 as uuidv4 } from 'uuid'
import { default as nodeCuid } from 'cuid'

export namespace npm {
  export async function uuid(ctx) {
    return uuidv4()
  }
  export async function cuid(ctx) {
    return nodeCuid()
  }
}

前端

import { $RPC } from './aca.client'

$RPC.server.npm.cuid()

# Aca.ts 是否可以只在前端运行？

Aca.ts 必须启动一个后端服务。前端接口实际上是访问后端的同名函数。所以不能只运行在前端。

# Aca.ts 如何创建多应用项目？

运行 aca server xxx 命令可以创建多个后端服务。aca client xxx 命令可以创建多个前端服务。每个服务都会生成相应的 API 接口。

注意： 数据库驱动类型为 Better-Sqlite3 时只能添加一个后端服务

  project
    .aca
    server1
    server2
    client1
    client2

# Aca.ts 是否可以在自建的前端或者后端服务中运行？

可以的。在自建的服务中运行 aca add -c 或者 aca add -s 就可以在自建服务中生成相应的 API

# 远程函数如何获取上下文？

函数的第一个参数为上下文，在生成前端的同名函数时，会把这个参数省去

# Aca.ts 是否会支持其他数据库？

会的，我们会进一步争对其他数据库进行适配。

# Aca.ts 生成的API不同数据库是否存在差异？

我们已经对不同数据库做了适配，不同的数据库生成的API都是相同的写法，开发者不需要进行修改。
